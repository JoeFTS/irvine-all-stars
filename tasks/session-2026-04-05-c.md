# Session Log — 2026-04-05-c (Tournament Discovery Cron Job)

## What Was Built

Nightly tournament discovery cron job that automatically finds SoCal youth baseball tournaments and adds them as drafts for admin review.

### Architecture
- **10-module Node.js ESM pipeline** at `scripts/tournament-discovery/`
- Runs nightly at 2am on Mac Mini (`ssh ana@100.100.46.56`) via PM2 cron
- Uses self-hosted Firecrawl (localhost:3002) for Google search + page scraping
- Inserts draft tournaments into `irvine_allstars.tournaments` (Supabase)
- Sends admin notification email via Resend to allstars@irvinepony.com

### Pipeline Flow
```
search → parse → filter → dedupe → images → insert → notify
```

### Files Created
```
scripts/tournament-discovery/
  config.mjs            - Search queries, aggregator URLs, division maps, city lists
  search.mjs            - Firecrawl /search and /scrape HTTP wrappers (60s timeout)
  parse.mjs             - Extract tournament data from markdown content
  filter.mjs            - Geographic + age/division + quality filtering
  dedupe.mjs            - Fuzzy name matching (Jaccard) + date proximity + intra-batch dedup
  images.mjs            - Download flyer images, upload to Supabase Storage
  insert.mjs            - Insert draft tournaments into DB (no announcements)
  notify.mjs            - Admin summary email via Resend
  discover.mjs          - Main orchestrator (entry point)
  ecosystem.config.cjs  - PM2 config (nightly 2am cron)
  .env                  - Secrets (Mac Mini only, not in git)
```

## Key Decisions

1. **Lives in the irvine-all-stars repo** under `scripts/` (versioned, easy to update via git pull)
2. **Direct HTTP to Firecrawl** (no SDK) — simpler for self-hosted instance
3. **Google search + aggregator direct scrapes** (TourneyMachine, USSSA, Triple Crown, Nations)
4. **Flyer images uploaded to Supabase Storage** (`tournament-flyers` bucket)
5. **PM2 cron on Mac Mini** (consistent with existing PM2 processes)
6. **No announcements created by script** — admin publishes tournament, auto-announce creates announcement

## Issues Found & Fixed During Deployment

### RLS Permission Error
- Service role key couldn't insert into `irvine_allstars.tournaments`
- Fix: `GRANT ALL ON ALL TABLES IN SCHEMA irvine_allstars TO service_role;`

### NULL Location Constraint
- Scraped tournaments sometimes have no parseable location
- Fix: Default `location` to `'TBD'` in insert.mjs (later superseded by quality gate)

### Announcements Went Live Immediately
- `announcements` table has no draft/published status — everything inserted is visible
- Fix: Removed announcement creation from discovery script entirely. Admin's publish action triggers auto-announce.

### From Email Address
- `noreply@irvineallstars.com` wasn't delivering
- Fix: Changed to `AllStars@irvineallstars.com` (verified domain in Resend, matches existing site emails)

### Junk Tournaments Inserted
- Old tournament pages (2024, 2025) and pages with no useful data were being inserted
- Fix: Added quality gate in filter.mjs requiring:
  1. Future start date (today or later)
  2. Real location (not empty/TBD)
  3. At least one of: registration URL, divisions, flyer, or meaningful description
- Deleted 4 junk draft tournaments from DB

### .env Loading
- PM2 doesn't natively support env_file
- Fix: Added zero-dependency .env loader in config.mjs (reads from script directory)
